### BERT

bert 改进，创新点，动机（问题、已有工作）。核心改进点、解决痛点。和 Bert 比较。




一、训练策略优化（不改动主体结构，“训得更好”）
1. RoBERTa（2019，Facebook）
	去掉 NSP（下一句预测），仅保留 MLM；
	动态掩码：每个 epoch 重新随机 mask，而非静态固定；
	更大 batch（8k→16k）、更多数据（160GB vs 13GB）、更长训练；
	Byte-level BPE，更稳处理 Unicode。
	效果：GLUE/SQuAD 显著超越原生 BERT。
2. SpanBERT（2019，CMU/Google）
	优化跨度 / 抽取类任务（QA、关系抽取）；
	引入跨度掩码 + 跨度边界预测，强化连续片段表征。
二、预训练目标革新（改任务，“学得更准”）
3. ELECTRA（2020，Google）
	不用 [MASK]，改用生成器 + 判别器（类 GAN）；
	小生成器生成 “合理替换词”，主模型判别每个 token 是否被替换；
	训练效率更高，同等算力下超越 MLM。
4. ALBERT（2019，Google）
	替换 NSP 为 SOP（句子顺序预测），更贴合语义连贯性；
	嵌入层参数分解（hidden 与 embedding 维度解耦）；
	跨层参数共享：12 层共享一套权重，参数量大减。
三、模型架构改进（改结构，“表达更强”）
5. DeBERTa（2020，微软）
	解耦位置与内容注意力：token 编码与位置编码分开算 attention；
	增强解码：用绝对位置补全相对位置信息；
	引入泛化正则，抑制过拟合。
6. XLNet（2019，CMU/Google）
	解决 BERT 的 [MASK] 预训练 / 微调不一致；
	用排列语言建模（PLM），自回归 + 双向上下文；
	引入双流注意力，处理位置依赖。
四、轻量化 / 压缩（小、快、省，“端侧能用”）
7. DistilBERT（2019，Hugging Face）
	知识蒸馏：BERT-base → 6 层学生模型；
	保留 60% 参数、95% 性能，推理快 60%。
8. TinyBERT（2019，华为）
	更激进蒸馏：4 层，仅 28% 参数；
	中间层 + 注意力矩阵对齐，精度损失小。
9. MobileBERT（2020，Google）
	深度可分离卷积 + 瓶颈结构；
	专为移动端优化，速度 / 内存友好。
五、长文本处理（突破 512 限制，“读得更长”）
10. Longformer（2020，Allen Institute）
	滑动窗口注意力：局部密集 + 全局稀疏；
	支持 4096 token 输入，文档级任务友好。
11. BigBird（2020，Google）
	块稀疏注意力：随机 + 局部 + 全局三部分；
	高效处理长序列，兼顾速度与性能。
六、知识增强 / 多模态（加外部信息，“理解更深”）
12. ERNIE（2019，百度）
	引入实体 / 知识图谱（如 “苹果 = 公司 / 水果”）；
	知识掩码 + 短语掩码，增强语义关联。
13. VL-BERT / Vision-BERT（多模态）
	文本 + 图像联合预训练；
	适配图像描述、VQA、多模态分类。
七、参数高效微调（PEFT，“微调更省”）
14. LoRA（2021，微软）
	冻结 BERT 主干，仅在注意力层加低秩矩阵；
	微调参数 <1%，性能接近全量微调。
15. Adapter Layers / Prefix-Tuning / BitFit
	Adapter：层间插小网络；
	Prefix-Tuning：只训输入前缀；
	BitFit：只训 bias 项；
	共同特点：少参高效、低成本适配下游任务。

|                | XLNet                            | 2019, CMU/Google      | 排列语言建模（PLM），自回归+双向上下文；双流注意力。                 | 核心灵感来自自回归语言模型（如GPT）和自编码模型（如BERT）的结合；借鉴排列语言建模和双流注意力（来自Transformer的掩码机制）。                               | BERT自编码模型存在预训练-微调不一致（掩码符号[MASK]在微调时不存在）；自回归模型只能单向建模。 |

|                | TinyBERT                         | 2019, 华为              | 更激进蒸馏，4层仅28%参数，中间层+注意力矩阵对齐。                  | 在DistilBERT基础上，进一步借鉴中间层对齐和数据增强（来自NLP数据增强技术）。                                                          | DistilBERT压缩程度不够，模型仍较大；蒸馏时仅关注输出层，忽略中间层知识。            |
|                | MobileBERT                       | 2020, Google          | 深度可分离卷积+瓶颈结构，专为移动端优化。                        | 灵感来自移动端模型压缩（如MobileNet的深度可分离卷积）和瓶颈结构（来自ResNet的瓶颈设计）。                                                  | 现有压缩模型仍不够轻量，无法在手机等移动设备上实时运行。                         |
| **五、长文本处理**    | Longformer                       | 2020, Allen Institute | 滑动窗口注意力（局部密集+全局稀疏），支持4096 token。             | 灵感来自稀疏注意力（如Sparse Transformer）和滑动窗口注意力（类似CNN的局部感受野），以及全局注意力（来自Transformer的全局交互）。                      | BERT的O(n²)注意力复杂度导致无法处理长文本（超过512 token）。              |
|                | BigBird                          | 2020, Google          | 块稀疏注意力（随机+局部+全局），高效处理长序列。                    | 借鉴图神经网络中的稀疏图结构（随机+局部+全局连接），以及块稀疏注意力（来自Sparse Transformer的块处理思想）。                                      | Longformer的滑动窗口注意力仍不够高效，且缺乏全局随机连接。                   |

|                | VL-BERT / Vision-BERT            | -                     | 文本+图像联合预训练，适配多模态任务。                          | 直接借鉴多模态学习（如ViLBERT、LXMERT）和视觉-语言联合预训练。                                                                | BERT仅处理文本，无法处理图像等多模态输入。                              |
