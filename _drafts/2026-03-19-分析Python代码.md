

## 附录 分析 Ruby 代码 


Ruby 本身没有直接“按行插装记录执行行”的内置方法，但可以通过 Ruby 的元编程特性、内置的追踪工具和代码解析能力来实现这个需求。下面我会给你介绍几种最实用的方案，从简单到进阶，都基于 Ruby 自带的功能。

### 方案 1：使用 `set_trace_func`（最简单，Ruby 内置）
`set_trace_func` 是 Ruby 最基础的执行追踪函数，能捕获代码执行的每一个事件（如行执行、方法调用、类定义等），可以直接用来记录执行过的代码行。

#### 完整示例代码
```ruby
# 定义追踪器：记录执行过的代码行
trace_log = []

# 设置追踪函数
set_trace_func lambda { |event, file, line, id, binding, classname|
  # 只关注 "line" 事件（代码行执行），过滤掉其他事件（如call/return等）
  if event == 'line'
    # 跳过追踪器自身的代码（避免无限递归）
    return if file == __FILE__ && line.between?(1, 20) # 根据实际行数调整

    # 读取当前执行行的代码内容
    begin
      code_line = File.readlines(file)[line - 1]&.strip
    rescue Errno::ENOENT
      code_line = "无法读取文件: #{file}"
    end

    # 记录关键信息：文件、行号、代码内容
    trace_log << {
      file: file,
      line: line,
      code: code_line,
      timestamp: Time.now
    }

    # 实时打印（也可以只存日志不打印）
    puts "[执行行] #{file}:#{line} | #{code_line}"
  end
}

# ========== 测试代码（你要追踪的逻辑）==========
def test_method
  a = 10
  b = 20
  c = a + b
  puts "结果: #{c}"
end

test_method

# ========== 停止追踪并输出日志 ==========
set_trace_func nil # 关闭追踪

puts "\n===== 执行记录汇总 ====="
trace_log.each do |log|
  puts "#{log[:file]}:#{log[:line]} - #{log[:code]}"
end
```

#### 代码解释
1. `set_trace_func` 接收一个 lambda 作为参数，每次代码触发事件时会调用这个 lambda；
2. `event == 'line'` 过滤出“代码行执行”事件（其他事件如 `call` 是方法调用、`return` 是方法返回）；
3. `File.readlines(file)[line - 1]` 读取对应文件的对应行内容（行号从 1 开始，数组索引从 0 开始）；
4. 最后用 `set_trace_func nil` 关闭追踪，避免影响后续代码。

#### 输出效果
```
[执行行] test.rb:22 | a = 10
[执行行] test.rb:23 | b = 20
[执行行] test.rb:24 | c = a + b
[执行行] test.rb:25 | puts "结果: #{c}"
结果: 30

===== 执行记录汇总 =====
test.rb:22 - a = 10
test.rb:23 - b = 20
test.rb:24 - c = a + b
test.rb:25 - puts "结果: #{c}"
```

### 方案 2：使用 `RubyVM::InstructionSequence`（进阶，解析代码行）
如果需要更精准地控制（比如只追踪特定方法/文件），可以用 `RubyVM::InstructionSequence`（Ruby 2.0+ 内置）解析代码的指令序列，结合 `eval` 插装每行代码。

#### 完整示例代码
```ruby
module LineTracer
  # 为指定方法插装行追踪
  def self.trace_method(klass, method_name)
    # 获取方法的原始代码
    method = klass.instance_method(method_name)
    file, start_line = method.source_location
    return unless file && start_line

    # 读取方法的所有代码行
    code_lines = File.readlines(file)[start_line - 1..-1]
    # 找到方法结束的行（简单处理：匹配 end 关键字）
    end_line = start_line
    code_lines.each_with_index do |line, idx|
      end_line = start_line + idx if line.strip == 'end'
      break if line.strip == 'end'
    end
    method_lines = File.readlines(file)[start_line - 1..end_line - 1]

    # 重新定义方法，每行都添加日志
    klass.class_eval do
      define_method(method_name) do |*args|
        method_lines.each_with_index do |line, idx|
          current_line = start_line + idx
          # 记录执行行
          puts "[执行行] #{file}:#{current_line} | #{line.strip}"
          # 执行当前行代码（跳过空行/注释）
          eval(line.strip) unless line.strip.empty? || line.strip.start_with?('#')
        end
      end
    end
  end
end

# ========== 测试 ==========
class Test
  def calculate
    x = 5
    y = 10
    z = x * y
    # 这是注释行
    puts "计算结果: #{z}"
  end
end

# 为 Test 类的 calculate 方法插装行追踪
LineTracer.trace_method(Test, :calculate)

# 执行方法
Test.new.calculate
```

#### 输出效果
```
[执行行] test.rb:35 | def calculate
[执行行] test.rb:36 | x = 5
[执行行] test.rb:37 | y = 10
[执行行] test.rb:38 | z = x * y
[执行行] test.rb:39 | # 这是注释行
[执行行] test.rb:40 | puts "计算结果: #{z}"
[执行行] test.rb:41 | end
计算结果: 50
```

### 方案 3：使用 `Coverage` 库（统计执行行，Ruby 2.5+ 内置）
如果你的需求是“统计哪些行被执行过”（而非实时记录），Ruby 2.5+ 内置的 `Coverage` 库是最佳选择，专门用于代码覆盖率统计，本质也是行级插装。

#### 完整示例代码
```ruby
require 'coverage'

# 启动覆盖率统计（:line 表示按行统计）
Coverage.start(:line)

# ========== 要追踪的代码 ==========
def foo
  a = 1
  b = 2
  if a > b
    puts "a 更大"
  else
    puts "b 更大"
  end
end

foo

# ========== 停止统计并输出结果 ==========
result = Coverage.result

# 解析结果：key 是文件路径，value 是 {行号: 执行次数}
result.each do |file, lines|
  puts "\n文件: #{file}"
  lines.each do |line_num, count|
    next if count.nil? # nil 表示该行不存在/未执行
    # 读取行内容
    code = File.readlines(file)[line_num - 1]&.strip
    puts "行 #{line_num}: #{count > 0 ? "执行过(#{count}次)" : "未执行"} | #{code}"
  end
end
```

#### 输出效果
```
b 更大

文件: test.rb
行 3: 执行过(1次) | def foo
行 4: 执行过(1次) | a = 1
行 5: 执行过(1次) | b = 2
行 6: 执行过(1次) | if a > b
行 7: 未执行 | puts "a 更大"
行 8: 执行过(1次) | else
行 9: 执行过(1次) | puts "b 更大"
行 10: 执行过(1次) | end
行 11: 执行过(1次) | end
行 14: 执行过(1次) | foo
```

### 关键注意事项
1. `set_trace_func` 会追踪**所有**代码行（包括 Ruby 内置库），需要过滤掉不需要的文件（如 Ruby 标准库路径）；
2. `Coverage` 库只统计“是否执行”和“执行次数”，无法实时记录执行顺序；
3. 插装会影响代码执行性能，仅用于调试/测试环境，不要在生产环境启用。

### 总结
1. **快速追踪执行行**：用 `set_trace_func`，Ruby 内置、无需额外依赖，适合简单场景；
2. **统计执行行覆盖率**：用 `Coverage` 库（Ruby 2.5+），专门用于行执行次数统计；
3. **精准插装特定代码**：用 `RubyVM::InstructionSequence` 解析代码，结合元编程自定义插装逻辑。

这三种方案都基于 Ruby 自带功能，无需安装任何 gem，可根据你的具体需求（实时记录/统计次数/精准控制）选择。
